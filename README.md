      <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-geofire-for-androidjava--realtime-location-queries-with-firebase" class="anchor" aria-hidden="true" href="#geofire-for-androidjava--realtime-location-queries-with-firebase"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>GeoFire for Android/Java — Realtime location queries with Firebase</h1>
<p>GeoFire is an open-source library for Android/Java that allows you to store and query a
set of keys based on their geographic location.</p>
<p>At its heart, GeoFire simply stores locations with string keys. Its main
benefit however, is the possibility of querying keys within a given geographic
area - all in realtime.</p>
<p>GeoFire uses the <a href="https://www.firebase.com/?utm_source=geofire-java" rel="nofollow">Firebase</a> database for
data storage, allowing query results to be updated in realtime as they change.
GeoFire <em>selectively loads only the data near certain locations, keeping your
applications light and responsive</em>, even with extremely large datasets.</p>
<p>A compatible GeoFire client is also available for <a href="https://github.com/firebase/geofire-objc">Objective-C</a>
and <a href="https://github.com/firebase/geofire-js">JavaScript</a>.</p>
<p>For a full example of an application using GeoFire to display realtime transit data, see the
<a href="https://github.com/firebase/geofire-java/tree/master/examples/SFVehicles">SFVehicles</a> example in
Android app in this repo.</p>
<h3><a id="user-content-integrating-geofire-with-your-data" class="anchor" aria-hidden="true" href="#integrating-geofire-with-your-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Integrating GeoFire with your data</h3>
<p>GeoFire is designed as a lightweight add-on to the Firebase Realtime Database. However, to keep things
simple, GeoFire stores data in its own format and its own location within
your Firebase database. This allows your existing data format and security rules to
remain unchanged and for you to add GeoFire as an easy solution for geo queries
without modifying your existing data.</p>
<h3><a id="user-content-example-usage" class="anchor" aria-hidden="true" href="#example-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Example Usage</h3>
<p>Assume you are building an app to rate bars and you store all information for a
bar, e.g. name, business hours and price range, at <code>/bars/&lt;bar-id&gt;</code>. Later, you
want to add the possibility for users to search for bars in their vicinity. This
is where GeoFire comes in. You can store the location for each bar using
GeoFire, using the bar IDs as GeoFire keys. GeoFire then allows you to easily
query which bar IDs (the keys) are nearby. To display any additional information
about the bars, you can load the information for each bar returned by the query
at <code>/bars/&lt;bar-id&gt;</code>.</p>

<p>GeoFire 2.x is based on the new 3.x release of <a href="https://firebase.google.com" rel="nofollow">Firebase</a>.</p>
<h3><a id="user-content-upgrading-from-geofire-10x-to-11x" class="anchor" aria-hidden="true" href="#upgrading-from-geofire-10x-to-11x"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Upgrading from GeoFire 1.0.x to 1.1.x</h3>
<p>With the release of GeoFire for Android/Java 1.1.0, this library now uses <a href="https://www.firebase.com/blog/2014-11-04-firebase-realtime-queries.html" rel="nofollow">the new query
functionality found in Firebase 2.0.0</a>.
As a result, you will need to upgrade to Firebase 2.x.x and add a new <code>.indexOn</code> rule to your
Security and Firebase Rules to get the best performance. You can view <a href="https://github.com/firebase/geofire-js/blob/master/examples/securityRules/rules.json">the updated rules
here</a>
and <a href="https://www.firebase.com/docs/security/guide/indexing-data.html" rel="nofollow">read our docs for more information about indexing your data</a>.</p>

<h2><a id="user-content-quickstart" class="anchor" aria-hidden="true" href="#quickstart"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Quickstart</h2>

<h3><a id="user-content-geofire" class="anchor" aria-hidden="true" href="#geofire"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>GeoFire</h3>
<p>A <code>GeoFire</code> object is used to read and write geo location data to your Firebase
database and to create queries. To create a new <code>GeoFire</code> instance you need to attach it to a Firebase database
reference.</p>
<div class="highlight highlight-source-java"><pre><span class="pl-smi">DatabaseReference</span> ref <span class="pl-k">=</span> <span class="pl-smi">FirebaseDatabase</span><span class="pl-k">.</span>getInstance()<span class="pl-k">.</span>getReference(<span class="pl-s"><span class="pl-pds">"</span>path/to/geofire<span class="pl-pds">"</span></span>);
<span class="pl-smi">GeoFire</span> geoFire <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">GeoFire</span>(ref);</pre></div>
<p>Note that you can point your reference to anywhere in your Firebase database, but don't
forget to <a href="https://github.com/firebase/geofire-js/blob/master/examples/securityRules">setup security rules for
GeoFire</a>.</p>
<h4><a id="user-content-setting-location-data" class="anchor" aria-hidden="true" href="#setting-location-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Setting location data</h4>
<p>In GeoFire you can set and query locations by string keys. To set a location for
a key simply call the <code>setLocation</code> method. The method is passed a key
as a string and the location as a <code>GeoLocation</code> object containing the location's latitude and longitude:</p>
<div class="highlight highlight-source-java"><pre>geoFire<span class="pl-k">.</span>setLocation(<span class="pl-s"><span class="pl-pds">"</span>firebase-hq<span class="pl-pds">"</span></span>, <span class="pl-k">new</span> <span class="pl-smi">GeoLocation</span>(<span class="pl-c1">37.7853889</span>, <span class="pl-k">-</span><span class="pl-c1">122.4056973</span>));</pre></div>
<p>To check if a write was successfully saved on the server, you can add a
<code>GeoFire.CompletionListener</code> to the <code>setLocation</code> call:</p>
<div class="highlight highlight-source-java"><pre>geoFire<span class="pl-k">.</span>setLocation(<span class="pl-s"><span class="pl-pds">"</span>firebase-hq<span class="pl-pds">"</span></span>, <span class="pl-k">new</span> <span class="pl-smi">GeoLocation</span>(<span class="pl-c1">37.7853889</span>, <span class="pl-k">-</span><span class="pl-c1">122.4056973</span>), <span class="pl-k">new</span> <span class="pl-smi">GeoFire</span>.<span class="pl-smi">CompletionListener</span>() {
    <span class="pl-k">@Override</span>
    <span class="pl-k">public</span> <span class="pl-k">void</span> <span class="pl-en">onComplete</span>(<span class="pl-smi">String</span> <span class="pl-v">key</span>, <span class="pl-smi">FirebaseError</span> <span class="pl-v">error</span>) {
        <span class="pl-k">if</span> (error <span class="pl-k">!=</span> <span class="pl-c1">null</span>) {
            <span class="pl-smi">System</span><span class="pl-k">.</span>err<span class="pl-k">.</span>println(<span class="pl-s"><span class="pl-pds">"</span>There was an error saving the location to GeoFire: <span class="pl-pds">"</span></span> <span class="pl-k">+</span> error);
        } <span class="pl-k">else</span> {
            <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(<span class="pl-s"><span class="pl-pds">"</span>Location saved on server successfully!<span class="pl-pds">"</span></span>);
        }
    }
});</pre></div>
<p>To remove a location and delete it from the database simply pass the location's key to <code>removeLocation</code>:</p>
<div class="highlight highlight-source-java"><pre>geoFire<span class="pl-k">.</span>removeLocation(<span class="pl-s"><span class="pl-pds">"</span>firebase-hq<span class="pl-pds">"</span></span>);</pre></div>

<div class="highlight highlight-source-java"><pre>geoFire<span class="pl-k">.</span>getLocation(<span class="pl-s"><span class="pl-pds">"</span>firebase-hq<span class="pl-pds">"</span></span>, <span class="pl-k">new</span> <span class="pl-smi">LocationCallback</span>() {

});</pre></div>
<h3><a id="user-content-geo-queries" class="anchor" aria-hidden="true" href="#geo-queries"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Geo Queries</h3>
<p>GeoFire allows you to query all keys within a geographic area using <code>GeoQuery</code>
objects. As the locations for keys change, the query is updated in realtime and fires events
letting you know if any relevant keys have moved. <code>GeoQuery</code> parameters can be updated
later to change the size and center of the queried area.</p>
<div class="highlight highlight-source-java"><pre><span class="pl-c"><span class="pl-c">//</span> creates a new query around [37.7832, -122.4056] with a radius of 0.6 kilometers</span>
<span class="pl-smi">GeoQuery</span> geoQuery <span class="pl-k">=</span> geoFire<span class="pl-k">.</span>queryAtLocation(<span class="pl-k">new</span> <span class="pl-smi">GeoLocation</span>(<span class="pl-c1">37.7832</span>, <span class="pl-k">-</span><span class="pl-c1">122.4056</span>), <span class="pl-c1">0.6</span>);</pre></div>
<h4><a id="user-content-receiving-events-for-geo-queries" class="anchor" aria-hidden="true" href="#receiving-events-for-geo-queries"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Receiving events for geo queries</h4>
<p>There are five kinds of events that can occur with a geo query:</p>
<ol>
<li><strong>Key Entered</strong>: The location of a key now matches the query criteria.</li>
<li><strong>Key Exited</strong>: The location of a key no longer matches the query criteria.</li>
<li><strong>Key Moved</strong>: The location of a key changed but the location still matches the query criteria.</li>
<li><strong>Query Ready</strong>: All current data has been loaded from the server and all
initial events have been fired.</li>
<li><strong>Query Error</strong>: There was an error while performing this query, e.g. a
violation of security rules.</li>
</ol>
<p>Key entered events will be fired for all keys initially matching the query as well as any time
afterwards that a key enters the query. Key moved and key exited events are guaranteed to be
preceded by a key entered event.</p>
<p>Sometimes you want to know when the data for all the initial keys has been
loaded from the server and the corresponding events for those keys have been
fired. For example, you may want to hide a loading animation after your data has
fully loaded. This is what the "ready" event is used for.</p>
<p>Note that locations might change while initially loading the data and key moved and key
exited events might therefore still occur before the ready event is fired.</p>
<p>When the query criteria is updated, the existing locations are re-queried and the
ready event is fired again once all events for the updated query have been
fired. This includes key exited events for keys that no longer match the query.</p>
<p>To listen for events you must add a <code>GeoQueryEventListener</code> to the <code>GeoQuery</code>:</p>
<div class="highlight highlight-source-java"><pre>geoQuery<span class="pl-k">.</span>addGeoQueryEventListener(<span class="pl-k">new</span> <span class="pl-smi">GeoQueryEventListener</span>() {
    <span class="pl-k">@Override</span>
    <span class="pl-k">public</span> <span class="pl-k">void</span> <span class="pl-en">onKeyEntered</span>(<span class="pl-smi">String</span> <span class="pl-v">key</span>, <span class="pl-smi">GeoLocation</span> <span class="pl-v">location</span>) {
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(<span class="pl-smi">String</span><span class="pl-k">.</span>format(<span class="pl-s"><span class="pl-pds">"</span>Key %s entered the search area at [%f,%f]<span class="pl-pds">"</span></span>, key, location<span class="pl-k">.</span>latitude, location<span class="pl-k">.</span>longitude));
    }

    <span class="pl-k">@Override</span>
    <span class="pl-k">public</span> <span class="pl-k">void</span> <span class="pl-en">onKeyExited</span>(<span class="pl-smi">String</span> <span class="pl-v">key</span>) {
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(<span class="pl-smi">String</span><span class="pl-k">.</span>format(<span class="pl-s"><span class="pl-pds">"</span>Key %s is no longer in the search area<span class="pl-pds">"</span></span>, key));
    }

    <span class="pl-k">@Override</span>
    <span class="pl-k">public</span> <span class="pl-k">void</span> <span class="pl-en">onKeyMoved</span>(<span class="pl-smi">String</span> <span class="pl-v">key</span>, <span class="pl-smi">GeoLocation</span> <span class="pl-v">location</span>) {
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(<span class="pl-smi">String</span><span class="pl-k">.</span>format(<span class="pl-s"><span class="pl-pds">"</span>Key %s moved within the search area to [%f,%f]<span class="pl-pds">"</span></span>, key, location<span class="pl-k">.</span>latitude, location<span class="pl-k">.</span>longitude));
    }

    <span class="pl-k">@Override</span>
    <span class="pl-k">public</span> <span class="pl-k">void</span> <span class="pl-en">onGeoQueryReady</span>() {
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(<span class="pl-s"><span class="pl-pds">"</span>All initial data has been loaded and events have been fired!<span class="pl-pds">"</span></span>);
    }

    <span class="pl-k">@Override</span>
    <span class="pl-k">public</span> <span class="pl-k">void</span> <span class="pl-en">onGeoQueryError</span>(<span class="pl-smi">DatabaseError</span> <span class="pl-v">error</span>) {
        <span class="pl-smi">System</span><span class="pl-k">.</span>err<span class="pl-k">.</span>println(<span class="pl-s"><span class="pl-pds">"</span>There was an error with this query: <span class="pl-pds">"</span></span> <span class="pl-k">+</span> error);
    }
});</pre></div>
<p>You can call either <code>removeGeoQueryEventListener</code> to remove a
single event listener or <code>removeAllListeners</code> to remove all event listeners
for a <code>GeoQuery</code>.</p>
<h4><a id="user-content-updating-the-query-criteria" class="anchor" aria-hidden="true" href="#updating-the-query-criteria"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Updating the query criteria</h4>
<p>The <code>GeoQuery</code> search area can be changed with <code>setCenter</code> and <code>setRadius</code>. Key
exited and key entered events will be fired for keys moving in and out of
the old and new search area, respectively. No key moved events will be
fired; however, key moved events might occur independently.</p>
<p>Updating the search area can be helpful in cases such as when you need to update
the query to the new visible map area after a user scrolls.</p>
